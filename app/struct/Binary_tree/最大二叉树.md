# LeetCode 654 最大二叉树：单调栈解法学习笔记

## 1 问题背景

**题目**：[LeetCode 654. 最大二叉树](https://leetcode.com/problems/maximum-binary-tree/)

给定一个不含重复元素的整数数组 `nums`。一个以此数组构建的最大二叉树定义如下：
- 树的根是数组中的最大元素。
- 左子树是通过数组中最大值左边部分构造出的最大二叉树。
- 右子树是通过数组中最大值右边部分构造出的最大二叉树。

要求返回构建的树的根节点。

**经典递归思路**：每次找到区间最大值，递归构建左右子树，时间复杂度 O(n²)。  
而这里要解析的是**利用单调栈实现的 O(n) 解法**。

---

## 2 代码实现

```cpp
class Solution {
public:
    TreeNode* constructMaximumBinaryTree(vector<int>& nums) {
        int len = nums.size();
        vector<int> stk;                 // 单调递减栈，存放下标
        vector<TreeNode*> tree(len);     // 每个下标对应的节点指针
        
        for (int i = 0; i < len; i++) {
            tree[i] = new TreeNode(nums[i]);

            // 弹出所有比当前元素小的栈顶元素
            while (!stk.empty() && nums[i] > nums[stk.back()]) {
                // 最后一个弹出的节点成为当前节点的左孩子
                tree[i]->left = tree[stk.back()];
                stk.pop_back();
            }

            // 如果栈非空，当前节点成为栈顶节点的右孩子
            if (!stk.empty()) {
                tree[stk.back()]->right = tree[i];
            }

            // 当前下标入栈
            stk.push_back(i);
        }

        // 栈底元素即为整棵树的根节点
        return tree[stk[0]];
    }
};
```

---

## 3 核心思想：单调递减栈

- **栈中保存的是已经遍历过的元素的下标**，且这些下标对应的值从栈底到栈顶是**严格递减**的。
- 遍历数组时，新元素 `nums[i]` 会与栈顶元素比较：
  - 若 `nums[i]` 大于栈顶元素，说明栈顶元素应该位于 `nums[i]` 的左子树中（因为它在左边且值更小），并且它是左子树中**最大的**（因为栈是递减的，越往下越大）。
  - 若 `nums[i]` 小于栈顶元素，则它应该成为栈顶元素的右孩子（因为它在右边且值更小）。
- 利用栈的这种“后进先出”特性，可以在一次遍历中确定每个节点的父节点和左右孩子关系。

---

## 4 详细步骤分解

1. 初始化空栈 `stk` 和节点数组 `tree`。
2. 遍历数组，对每个下标 `i`：
   - 创建节点 `tree[i]`，值为 `nums[i]`。
   - **处理左孩子**：  
     while 循环弹出所有值小于 `nums[i]` 的栈顶元素，并将最后一个弹出的节点设置为 `tree[i]` 的左孩子。  
     → 因为弹出的节点都是位于 `i` 左边且值小于 `nums[i]` 的，它们都应该在 `nums[i]` 的左子树中，而最后一个弹出的节点是这些节点中最大的（栈底方向更大），所以它成为左子树的根，即 `tree[i]` 的左孩子。
   - **处理右孩子**：  
     若 while 结束后栈非空，说明栈顶元素的值大于 `nums[i]`，且它在 `i` 左边。那么 `tree[i]` 应该成为它的右孩子（因为 `i` 在它右边且值更小）。
   - 将当前下标 `i` 压入栈中，保持栈的递减性。
3. 遍历结束后，栈中剩下的元素都是从未成为别人右孩子的节点（即它们没有父节点），其中栈底元素对应整个数组的最大值，即为根节点。

       / \

       \   \
        2   0

符合最大二叉树的定义。

## 6 关键点解析

### 6.1 为什么使用单调递减栈？
- 最大二叉树要求：**每个节点的父节点是它左右两边第一个比它大的元素**。这与**单调栈**寻找“下一个更大元素”的模式高度吻合。
- 单调递减栈保证了栈中元素从底到顶是递减的，这样新元素与栈顶比较时，可以快速确定：
  - 新元素大于栈顶 → 栈顶元素应位于新元素的左子树。
  - 新元素小于栈顶 → 新元素应位于栈顶元素的右子树。

### 6.2 节点连接的覆盖问题
- 在 while 循环中，`tree[i]->left` 可能会被多次赋值。这是因为每次弹出的节点都是当前栈顶（值较小），而最后一个弹出的节点才是所有小于 `nums[i]` 的节点中最大的那个，因此最终左孩子就是正确的。
- 类似地，`tree[stk.back()]->right` 也可能被后续元素覆盖。例如节点 6 的右孩子先设为 0，后又被 5 覆盖，最终保留了正确的较大值 5。这种覆盖是合理的，因为后出现的元素如果更大，就应该取代之前的右孩子位置。

### 6.3 根节点的确定
- 遍历结束后，栈中剩余的元素都是那些从未成为别人右孩子的节点（即它们没有父节点）。其中**栈底**元素是遍历过程中遇到的**最大值**，且它没有父节点，因此它就是整棵树的根。
- 即使最大值不在数组开头，由于栈的单调递减性，它一定会被保留到栈底。

---

## 7 复杂度分析

- **时间复杂度**：每个元素入栈一次，出栈最多一次，while 循环总执行次数不超过 n，因此总体为 **O(n)**。
- **空间复杂度**：栈和节点数组各 O(n)，节点空间 O(n)，总空间 **O(n)**。

相比递归法的 O(n²)，单调栈法在时间上有显著提升。

---

## 8 总结与拓展

- 单调栈是一种非常强大的工具，常用于处理“下一个更大/更小元素”类问题。在构建最大二叉树时，它帮助我们隐式地确定了每个节点的父节点。
- 这种构建方式也可以看作是一种**笛卡尔树**的构建过程（对数组按索引中序遍历得到原序列，且满足堆性质：父节点值大于子节点）。
- 掌握这一解法有助于理解单调栈的灵活应用，以及如何通过线性扫描构建树形结构。
