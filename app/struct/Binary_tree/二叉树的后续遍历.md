# LeetCode 145. 二叉树的后序遍历 —— 学习笔记

## 1 题目概述

**题目**：[LeetCode 145. 二叉树的后序遍历](https://leetcode.cn/problems/binary-tree-postorder-traversal/description/)

给你一棵二叉树的根节点 `root`，返回其节点值的 **后序遍历** 结果（以列表形式返回）。

**后序遍历顺序**：对于树中的每个节点，先访问它的**左子树**，再访问它的**右子树**，最后访问该**节点本身**（即 **左 → 右 → 根**）。

**示例**：
- 输入：`root = [1, null, 2, 3]`
- 输出：`[3, 2, 1]`

**提示**：
- 节点数范围 `[0, 100]`
- 节点值范围 `[-100, 100]`

**进阶**：递归算法很简单，你可以通过迭代算法完成吗？

## 2 迭代解法代码

```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left), right(right) {}
 * };
 */
class Solution {
public:
    vector<int> postorderTraversal(TreeNode* root) {
        vector<int> ans;                // 存储遍历结果
        stack<TreeNode*> stk;            // 辅助栈
        TreeNode* pred = nullptr;        // 记录上一个访问的节点

        while (root || !stk.empty()) {
            // 1. 一路向左，将当前节点的所有左子节点入栈
            while (root) {
                stk.push(root);
                root = root->left;
            }

            // 2. 查看栈顶节点
            root = stk.top();

            // 3. 判断是否可以访问当前节点
            //    条件：没有右子树 或者 右子树已经被访问过（即上一次访问的节点是右孩子）
            if (!root->right || root->right == pred) {
                ans.push_back(root->val);   // 访问当前节点
                stk.pop();                   // 弹出
                pred = root;                  // 更新上一个访问节点
                root = nullptr;                // 将root置空，下一轮从栈中取新节点
            } else {
                // 否则，先处理右子树
                root = root->right;
            }
        }
        return ans;
    }
};
```

## 3 算法思路解析

### 3.1 核心思想
后序遍历的难点在于：**在访问一个节点之前，必须保证它的左右子树都已访问完毕**。使用栈模拟时，我们无法像前序或中序那样简单地根据栈顶决定是否访问，因为即使栈顶节点存在，它的右子树可能尚未处理。

因此，需要一个额外的指针 `pred` 来记录**上一次被访问的节点**，从而判断当前节点的右子树是否已经处理完：
- 如果当前节点**没有右子树**，则可以安全访问。
- 如果当前节点**有右子树**，且 `pred` 正好指向它的右孩子，说明右子树已经访问完毕，也可以安全访问。
- 否则，需要先转向右子树进行处理。

### 3.2 算法步骤分解

1. **初始化**：结果数组 `ans` 为空，栈 `stk` 为空，前驱指针 `pred = nullptr`。
2. **外层循环**：当 `root` 不为空或栈不为空时，重复执行。
3. **左链入栈**：内层 `while` 循环将当前节点 `root` 及其所有左子节点依次入栈，直到 `root` 为空。这一步保证了我们总是先处理最左侧的节点。
4. **查看栈顶**：将 `root` 指向栈顶节点（但不弹出），准备判断。
5. **判断访问条件**：
   - 如果当前节点**没有右子树** 或 **右子树已被访问**（`root->right == pred`）：
     - 记录当前节点值到 `ans`。
     - 从栈中弹出该节点。
     - 更新 `pred` 为当前节点。
     - 将 `root` 置为 `nullptr`，以便下一轮循环继续从栈中取下一个节点。
   - 否则（存在右子树且未被访问）：
     - 将 `root` 指向其右孩子，下一轮循环将进入该右子树的处理流程。
6. **循环结束**，返回 `ans`。

### 3.3 示例模拟（`root = [1, null, 2, 3]`）

- **初始**：`root` 指向节点1，栈空，`pred = nullptr`。

| 轮次 | 操作前状态 (`root`, 栈内容) | 操作                                                                                                     | 操作后状态                  | 说明       |
| ---- | --------------------------- | -------------------------------------------------------------------------------------------------------- | --------------------------- | ---------- |
| 1    | root=1, 栈=[]               | 左链入栈 → 栈=[1]；root=栈顶1；判断有右子树且未访问 → root=右孩子2                                       | root=2, 栈=[1]              | 转向右子树 |
| 2    | root=2, 栈=[1]              | 左链入栈 → 节点2、节点3依次入栈 (栈=[1,2,3])；root=栈顶3；判断无右子树 → 访问3，弹出3，pred=3，root=null | root=null, 栈=[1,2], pred=3 | 访问节点3  |
| 3    | root=null, 栈=[1,2], pred=3 | root=栈顶2；判断无右子树 → 访问2，弹出2，pred=2，root=null                                               | root=null, 栈=[1], pred=2   | 访问节点2  |
| 4    | root=null, 栈=[1], pred=2   | root=栈顶1；判断有右子树且 `root->right == pred` (2) → 访问1，弹出1，pred=1，root=null                   | root=null, 栈=[], pred=1    | 访问节点1  |
| 5    | root=null, 栈=[]            | 循环结束                                                                                                 |                             |            |

最终结果：`[3, 2, 1]` ✅

## 4 复杂度分析

- **时间复杂度**：**O(n)**，其中 n 是二叉树的节点数。每个节点恰好被压入栈一次、弹出栈一次，并且被访问一次。
- **空间复杂度**：**O(n)**。栈在最坏情况下（树退化为一条链）需要存储所有节点。平均情况下为 O(log n)（树的高度）。

---

## 5 总结与要点

- **后序遍历的迭代实现关键在于判断何时可以访问节点**，需要额外记录上一个访问的节点，以确认右子树是否处理完毕。
- 使用 `pred` 指针巧妙地解决了“回溯时无法获知右子树状态”的问题。
- 该方法同样适用于中序和前序遍历的变种，只需调整访问时机和压栈顺序即可。
- 理解此解法有助于深入掌握栈在二叉树遍历中的应用，是面试和算法竞赛中的常见考点。
